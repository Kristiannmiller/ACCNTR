# ContentFeedForHealthNews

Story Link: [#360](https://github.com/ahc-digital/migration-planning-tracking/issues/360)

Description: The content feed with card creates a list of cards from the latest blog articles with the ability to point at an external feed for creating the cards.

---
# Acceptance Criteria
The component is created within Sitecore and meets the following criteria:

- Template name and structure matches specification in "Sitecore Details" section
- Content authors are able to add, modify, and remove the component via experience editor
- For ADA, component should be fully operable via keyboard (tab index, panel toggling, etc) without mouse interaction. Component must also pass Storybook ADA plugin validation.

Component is added to storybook, and each content author editable field is editable within storybook.

Should implement the exclude from translation class around the entire component.

The top of the container should be the anchor target with the id from the rendering parameter.

The list of articles will be created automatically from the Healthy News API. The slug from the rendering parameters should be passed to the API to help filter the results.

Each card contains: a title, support copy (max length of 122 characters, after this it's truncated with "..."), and an image. All content in the card is automatically left aligned. The entire card is clickable.

On desktop the grid is 4 columns. On tablet, the grid is 2 columns. On mobile the cards are stacked or in a carousel.

Images may be hidden for all cards on mobile.

Support copy may be hidden for all cards on all viewport sizes.
