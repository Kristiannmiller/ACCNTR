# Form

Story Link: [#162](https://github.com/ahc-digital/component-library/issues/162)

Description: The form component is responsible for rendering out a form from Form.io

---
# Acceptance Criteria
The component is created within Sitecore and meets the following criteria:

Template name and structure matches specification in "Sitecore Details" section
Content authors are able to add, modify, and remove the component via experience editor
For ADA, component should be fully operable via keyboard (tab index, panel toggling, etc) without mouse interaction. Component must also pass Storybook ADA plugin validation.

Component is added to storybook, and each content author editable field is editable within storybook.

The target Form.io environment can be configured via a site settings item.

<Stories />

## Props
<Controls />

- formName
  - Form.io form name that you want rendered 

- thankYouPageUrl
  - Redirect info to the thank you page applied after successful submit 

- formioDomain
  - The Form.io environment that your desired form is in 


## Example Usage(s)
```tsx
<Form
  formName={<Text field={{value: 'ahcContactUs'}} />}
  formioDomain={<Text field={{value: 'live'}} />}
  thankYouPageUrl={<Link field={{href: '/thanks-for-submission/'}} />}
/>
```
