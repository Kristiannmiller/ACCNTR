# Overview
This project is designed to provide a comprehensive solution for building and maintaining a React Component Library.
It includes a collection of reusable components that can be easily integrated into any React application.

## Technologies Used
- **Commitlint**: A tool that checks if the commit messages meet the conventional commit format.
- **CVA**: A tool that helps manage classnames based on props.
- **ESLint**: A tool for identifying and reporting on patterns found in ECMAScript/JavaScript code.
- **FontAwesome**: A popular icon set and toolkit that provides a comprehensive collection of icons.
- **Husky**: A tool that makes it easy to use githooks as if they are npm scripts.
- **Jest**: A delightful JavaScript Testing Framework with a focus on simplicity.
- **Lint-Staged**: A tool that runs linters on git staged files.
- **Plop**: A micro-generator that helps create new files and folders from templates.
- **Prettier**: An opinionated code formatter that enforces a consistent code style.
- **Radix UI**: A library of unstyled, accessible components for building high-quality design systems and web apps.
- **React Testing Library**: A simple and complete testing library for React that encourages good testing practices.
- **React**: A JavaScript library for building user interfaces.
- **SCSS**: A CSS preprocessor for writing more maintainable and scalable styles.
- **ShadCN UI**: A headless React Component Library for building design systems with a focus on accessibility and developer experience that wraps Radix UI.
- **Storybook**: An open-source tool for developing UI components in isolation for React, Vue, and Angular.
- **Swiper**: A modern touch slider that is focused on performance and best practices.
- **TypeScript**: A typed superset of JavaScript that compiles to plain JavaScript.
- **Vite**: A build tool that aims to provide a faster and leaner development experience for modern web projects.
